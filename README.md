## Quick start (first week)

### Monday-Friday
- **10 min morning**: Review one LeetCode problem category
- **1 hour evening** (non-office days): Solve 1 LeetCode problem
- **Office days**: 30 min late night — one DSA flashcard or recruiter message

### Weekend
- **Sunday evening**: Fill in `weekly_log.md`
- **Friday evening**: Push solutions to GitHub and update `leetcode_checklist.md`

**How to push:**
1. Solve problem on LeetCode
2. Write solution in Java in `solutions/[Category]/[ProblemName].java`
3. Add comments: approach, time complexity, space complexity
4. Commit with message: `[Easy/Medium/Hard] Problem Name - Solution`
5. Push every Friday

## Key metrics to track

### Monthly targets
- **LeetCode**: 20-30 problems/month (80 total)
- **System Design**: 1 case study/month (4 total)
- **Applications**: 100+ per month (25/week)
- **Interviews**: 1-2 in month 1, 3-4 in month 2, 5+ in month 3
- **Offers**: target 1 offer by end of month 3

### Success indicators
- 10-15% interview callback rate on applications
- Able to explain one system design in 10 minutes clearly
- Can solve medium LeetCode problems in <30 min
- Received at least 2 offers in the 3-month window

---

## Adjustments based on progress

### If DSA is too slow
- Reduce weekly problems by 1-2
- Focus on one category at a time
- Watch video walkthroughs before solving

### If no interview callbacks
- Refine resume keywords (use JobScan)
- Add more cloud projects to GitHub
- Personalize cover letters better
- Reach out to recruiters directly

### If system design feels hard
- Watch 1-2 YouTube walkthroughs first
- Start with simple design, expand gradually
- Practice explaining to a friend

---

## Important mindset

1. **Progress over perfection**: solving problems imperfectly is better than not solving
2. **Consistency beats intensity**: 1 hour daily > 10 hours once a week
3. **Apply often**: 25+ applications/week is not spamming, it's statistics
4. **Iterate fast**: if something doesn't work, change it in 1 week, don't wait 3 months
5. **Office days are OK to be light**: remote days are when heavy lifting happens

---

## Questions or blockers?

If anything feels unclear or too much:
1. Scale down: solve fewer problems, apply to fewer roles
2. Pick one focus: DSA for 2 weeks, then system design for 2 weeks
3. Review `resources.md` for supplementary learning

**Remember**: You have 4 years of real experience. You're not starting from zero. 
This plan is about **showing it** through interview skills and **moving to a market** that values it.
