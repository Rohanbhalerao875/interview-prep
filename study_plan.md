# Study Plan for International / Remote Job Transition

## Objective
- Move from 13.25 LPA to 30-40 LPA
- Target remote or international roles
- Maintain progress despite 10 office commute days per month

## Weekly rhythm

### Office days (Tue/Wed/Thu + alternate Wed)
- Leave 9:30 AM, return 11 PM–12 AM
- Use late night for one small accomplishment:
  - 1 DSA flashcard or problem review
  - 1 system-design note
  - 1 LinkedIn/recruiter message
  - 1 small career admin task

### Non-office days
- Morning: 2 hours focused study
- Evening: 1.5–2 hours practice
- Aim for:
  - 1-2 coding problems (LeetCode)
  - 1 project/cloud task
  - 1 application or networking action

## Priority focus (in order)
1. **DSA practice via LeetCode** — track in `leetcode_checklist.md`
2. **Java + Spring backend fundamentals** — core strength
3. **System design case studies** — document in `system_designs.md`
4. **Cloud practical work** — AWS/GCP/Azure hands-on
5. **Resume, LinkedIn, and applications** — tracked in `job_applications.md`

## Monthly milestones
- **Month 1 (Jun 25 – Jul 25)**: establish routine, resume refresh, one cloud project plan, solve 20 DSA problems
- **Month 2 (Jul 26 – Aug 25)**: reach 60 DSA problems, complete 2 system designs, start interview outreach
- **Month 3 (Aug 26 – Sep 25)**: active interviews, feedback-driven improvements, 80+ problems solved

## Daily "done" checklist
- [ ] Solve or review one problem (LeetCode)
- [ ] Read a short note or listen to tech/audio lesson
- [ ] Do one application / outreach / profile update

## Notes
- Use commute time for audio learning and note-taking
- Keep the plan simple, small wins every day
- Update `weekly_log.md` every Sunday
- GitHub push every Friday with solved problems and documentation
- Follow the review policy in `skills/dsa-review-policy.md` for all DSA updates
