### Primary platform
- **LeetCode**: https://leetcode.com

  - Focus: Arrays, Strings, Trees, Graphs, DP (in that order)
  - Target: 80 problems in 3 months (see `leetcode_checklist.md`)

- **GeeksforGeeks**: https://www.geeksforgeeks.org/ (free + paid)

- **HackerRank**: https://www.hackerrank.com/ (interactive challenges)
- **CodeSignal**: https://codesignal.com/ (game-based learning)

### YouTube channels
- TakeUForward (DSA in Hindi, excellent clarity)
- Tushar Roy (graph and DP explanations)
- Kevin Naughton Jr. (LeetCode walkthroughs)

## System Design

### Books (summary versions online)
- "Designing Data-Intensive Applications" by Martin Kleppmann
- "System Design Interview" by Alex Xu (two-volume set)
- "Fundamentals of System Design" courses

### YouTube & Courses
- Grokking System Design Interview (Educative.io)
- System Design Interview (Tech Dummies Narendra L)
- ByteByteGo (Alex Xu's YouTube channel)
- Gaurav Sen (System Design Primer)

### Key topics to master
1. Scalability and load balancing
2. Caching strategies (Redis)
3. Database design and sharding
4. Message queues (Kafka, RabbitMQ)
5. Microservices architecture
6. API design and versioning
7. Monitoring and logging
8. Disaster recovery and failover

## Cloud Platforms

### AWS (recommended for your goals)
- Free tier: https://aws.amazon.com/free/
- Services to learn:
  - EC2 (compute)
  - RDS (databases)
  - S3 (storage)
  - API Gateway
  - Lambda (serverless)
  - ElastiCache (Redis)
  - SNS/SQS (messaging)

- Certifications:
  - AWS Certified Developer Associate (most relevant)
  - AWS Certified Solutions Architect Associate

### Course options
- A Cloud Guru (AWS courses)
- Linux Academy (AWS and DevOps)
- Udemy (affordable, 80-90% discounts often available)
- Coursera (free audit option)

### Hands-on projects
1. Deploy a Spring Boot app to EC2
2. Use RDS for database
3. Add S3 for file storage
4. Set up CloudFront CDN
5. Use IAM for security

## Behavioral Prep

### STAR story framework
- **Situation**: Context of the challenge
- **Task**: What needed to be done
- **Action**: What you did
- **Result**: Quantified outcome

### Common questions & stories to prepare
1. Tell me about a system you designed.
2. How do you approach a performance issue?
3. Tell me about a time you disagreed with a manager.
4. Describe your most complex project.
5. How do you handle tight deadlines?
6. Tell me about a technical challenge you overcame.

### Practice
- Record yourself answering
- Time yourself (2-3 min per story)
- Share with friends for feedback
